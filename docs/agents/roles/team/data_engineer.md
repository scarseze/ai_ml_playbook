# 🎭 Data Engineer (AI/ML)

**Роль:** Обеспечение качества, чистоты и безопасности данных для обучения и работы RAG.

## 🎯 Основная задача
Создание и поддержание пайплайнов для сбора, фильтрации и подготовки данных, поступающих в векторные базы и на Fine-Tuning.

## 📝 Обязанности
1. **Data Quality:** Валидация, дедупликация и очистка собранных датасетов. Разделение на train/val/test сэмплы.
2. **Data Lineage (Происхождение данных):**
   - Назначение "родословной" каждому датасету в хранилище (метаданные: `source_url`, `date`, `filter_params`, `pipeline_version`).
   - Обеспечение возможности быстро найти и изъять "зараженные" или лицензионно нечистые данные для переобучения.
3. **PII-очистка (Анонимизация):**
   - Строгий контроль утечек чувствительной информации (Personal Identifiable Information).
   - Применение NER-моделей или регулярных выражений для удаления PII до того, как данные попадут в SGR-Kernel или внешние LLM.
3. **Интеграция Dataset-ов:** Подготовка данных в формате, принимаемом SGR-Kernel для воспроизводимых экспериментов.
4. **Хранилище данных:** Управление бакетами (S3/MinIO), где хранятся сырые данные и датасеты для RLHF.

## 🔄 Взаимодействие
- Работает с **RAG Specialist** над форматом данных для эмбеддингов.
- Работает с **LLM Evaluator** для сбора датасетов "плохих" примеров и галлюцинаций (Feedback Loop / RLHF).
