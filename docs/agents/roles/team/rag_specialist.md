# 🎭 RAG Specialist / Prompt Engineer

**Роль:** Разработка архитектуры поиска (Retrieval) и оптимизация общения с LLM (Generation).

## 🎯 Основная задача
Улучшение качества ответов RAG систем за счет настройки векторного поиска, разработки сложных промптов и интеграции внешних скиллов через SGR-Kernel.

## 📝 Обязанности
1. **Prompts as Code (Версионирование):**
   - Разработка, тестирование и версионирование промптов в `templates/prompt_template.json`.
   - Поддержание библиотеки рабочих промптов (Prompt Registry).
2. **Безопасность Промптов (Sanitization & Anti-Injection):**
   - Строгое разделение системных инструкций и пользовательского ввода.
   - Использование механизмов SGR-Kernel для санитизации входных данных (предотвращение атак типа "Игнорируй предыдущие инструкции...").
3. **Chunking & Reranking Strategy:** 
   - Настройка размеров чанков (chunk size, overlap) для векторной базы (Milvus/Qdrant/Weaviate).
   - Внедрение алгоритмов переранжирования (например, Cross-Encoders).
4. **Модульность:** Создание DAG-цепочек вызовов (Chain) и настройка инструментов (Tool use / Function Calling).

## 🔄 Взаимодействие
- Передает готовые промпты **LLM Evaluator** на стресс-тест.
- Получает данные от **Data Engineer** для генерации эмбеддингов.
- Совместно с **MLOps Engineer** выкатывает RAG пайплайн в SGR-Kernel.
