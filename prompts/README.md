# Директория промптов (Prompts as Code)

Добро пожаловать в репозиторий знаний нашего RAG и LLM взаимодействия.
Здесь хранятся все системные (System) и пользовательские (User/Few-shot) промпты.

## 📌 Главное правило
**Любой промпт здесь — это продуктовый код.** Вы не меняете промпты в UI интерфейсах OpenAI или Anthropic (playground), вы меняете их в этой директории, тестируете и отправляете Pull Request.

## 🗂 Структура
Рекомендуется разбивать промпты по доменам или названиям пайплайнов (например: `rag_system/`, `evaluation/`, `sql_agent/`).
Структуру промпта наследовать из эталона: `../templates/prompt_template.json`.

## 🔄 Жизненный цикл промпта
1. **Создание/Обновление:** `RAG Specialist` создает новую версию (например `v1.2.0`).
2. **Evaluation:** Промпт проходит обязательный стресс-тест в `SGR-Kernel-R7` (Continuous Evaluation, Red Teaming) через `LLM Evaluator` агента.
3. **Review:** Изменения добавляются в `main` через PR *(см. `pull_request_template.md`)*.
4. **Deploy:** `MLOps Engineer` подхватывает изменения и регистрирует их в системе.

## 🏷️ Версионирование
Рекомендуется SemVer:
- `v1.0.x` - Мелкие фиксы пунктуации или форматирования ответа (без изменения сути).
- `v1.x.0` - Добавление новых инструкций, Capability или Few-shot примеров.
- `vX.0.0` - Полная переработка логики работы агента, смена Target архитектуры или модели (например: переход с `GPT-3.5` на `Llama-3`).
